# Task 105：reset password 验证码发送按钮展示逻辑

## 任务内容

`reset password` 密码弹框和 `signin-reset-password.html` 页面中，验证码区域需要先展示发送验证码按钮，点击以后才展示倒计时。手机版和 pc 版都需要修改。

## 修改内容

- 修改 `src/partials/signin-modal.html`：
  - 将 reset password 验证码区域从默认展示 `30s` 调整为默认展示 `Send` 按钮。
  - 保留倒计时文本节点，点击发送后再展示倒计时。
- 修改 `js/login.js`：
  - 新增验证码发送按钮初始化逻辑。
  - 点击 `Send` 后切换到 `30s` 倒计时，倒计时结束恢复 `Send`。
  - 每次进入 reset password 视图时重置发送状态，避免上次倒计时残留。
- 修改 `css/signin.less` 与 `css/signin.css`：
  - 新增 `Send` 按钮样式和倒计时显示/隐藏状态。
  - 增加验证码输入框右侧留白，避免按钮或倒计时覆盖输入内容。
- 执行 `node scripts/build-pages.js`：
  - 同步生成根目录页面产物。
  - 刷新缓存版本为 `20260613080246`。

## 验证

- `node --check js/login.js`
- `node scripts/build-pages.js`

## 影响页面

- 公共登录弹框：所有包含 `src/partials/signin-modal.html` 的页面
- 独立重置密码页面：`signin-reset-password.html`
- 由于公共弹框被全站页面复用，根目录 HTML 产物通过构建统一刷新缓存版本。
