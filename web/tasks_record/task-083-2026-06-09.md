# Task 083：多个页面手机版二级菜单点击高亮移除

## 任务内容

多个页面手机版左上角菜单中，进入 Design、Trading 等二级菜单页后，点击任意选项不应出现 `Trading 2` 上的蓝色矩形背景。

## 修改页面

- 所有使用公共左上角菜单的页面
- 以 `post-detail.html` 为主要验证页面

## 修改内容

- 在 `js/common.js` 中为手机版二级菜单增加触摸与点击防护：
  - 拦截二级菜单标题和条目的默认触摸高亮、文本选择和事件冒泡。
  - 点击二级菜单链接时由脚本接管跳转，保留原有链接跳转能力。
  - 菜单打开时标记 `home-menu-open`，用于清理二级菜单内的文本选择。
- 在 `css/common.less` 的手机版媒体查询内增加二级菜单防高亮样式，并同步更新 `css/common.css`：
  - 禁用 `-webkit-tap-highlight-color`、文本选择和 touch callout。
  - 清除二级菜单标题、条目在 active/focus/selection 下的背景和 outline。
- 通过 `node scripts/build-pages.js` 重新生成根目录静态页面，刷新资源版本防缓存。

## 验证

- `node --check js/common.js` 通过。
- `node scripts/build-pages.js` 通过，最终资源版本为 `20260609002611`。
- 尝试调用 chrome mcp 验证时，`list_pages` 在 120 秒后超时，工具层未返回可用页面。
- 使用本机 Chrome Headless 的 DevTools Protocol 按一加 Ace5 Pro 等效视口 `393x873`、DPR `2.75`、Android UA 验证：
  - `Trading` 二级菜单正常展示。
  - 触摸按下 `Trading 21` 时，`Trading 2` 标题、条目和父级容器背景均为透明。
  - `-webkit-tap-highlight-color` 均为透明，未产生选中文本。
  - 二级菜单条目点击后仍会跳转到原链接。
