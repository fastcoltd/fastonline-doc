# Task 030 - 2026-05-19

## 任务
点击 `index.html` 右上角 Join 出现的一系列弹框（登录/注册/找回密码/2FA）在手机版支持向上滚动，确保可滚动到最底部完整展示内容。

## 修改页面
- `index.html`（由 `src/pages/index.html` 构建产物）

## 变更摘要
- 修复 Join 弹框打开时的滚动锁冲突：
  - `js/login.js` 中为该弹框场景增加 `signin-modal-open` 标记类。
  - `css/common.less` / `css/common.css` 中对 `body.modal-open.signin-modal-open` 放开纵向触摸滚动（`touch-action: pan-y`）。
- 调整移动端弹框滚动容器策略（仅移动端）：
  - `css/signin.less` / `css/signin.css` 将 `signin-bg` 作为主滚动容器（`overflow-y: auto`、`height: 100dvh`、`touch-action: pan-y`）。
  - `signin-container` 及登录/注册/2FA 容器改为 `height: auto` + `min-height: 100%`，避免安卓机型双滚动容器导致手势失效。
- 新增防缓存机制，确保你每次访问都拿到最新资源：
  - `scripts/build-pages.js` 自动给本地 `css/*.css`、`js/*.js` 添加构建时间戳 `?v=...`。
  - `scripts/build-pages.js` 自动注入 no-cache meta（`Cache-Control/Pragma/Expires`）。
  - 已执行 `node scripts/build-pages.js`，根目录产物已更新。

## 影响范围
- 仅移动端 Join 弹框滚动行为与资源缓存策略。
- 不影响 PC 端布局与交互。
