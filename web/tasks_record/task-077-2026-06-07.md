# Task 077：item-detail.html 手机版菜单打开页面不右移

## 任务内容

item-detail.html 页面的手机版需要修改。点击左上角按钮出现菜单时，底层 item-detail.html 页面整体会向右移动一些，需要出现菜单时底部页面整体不要移动。另外检查其他会出现这个菜单的页面，修改后所有会出现这个菜单的页面，出现菜单后页面内容都不会移动。

## 修改页面

- `item-detail.html`
- 所有复用公共 `home-menu` 菜单的根目录页面产物

## Diff 总结

- 在 `js/common.js` 中，为左上角移动端菜单打开状态增加 `body.home-menu-open` 标记，关闭菜单时同步移除。
- 在 `css/common.less` 与 `css/common.css` 中，为移动端 `body.modal-open.home-menu-open` 保留原有 `padding-right: 2rem`，避免公共 `modal-open` 滚动锁定覆盖右侧 padding 后导致页面内容横向位移。
- 公共菜单逻辑生效于 `item-detail.html` 以及其他复用 `home-menu` 的页面，PC 端不受影响。
- 运行 `node scripts/build-pages.js` 重新生成根目录页面，防缓存版本更新为 `20260607125810`。
