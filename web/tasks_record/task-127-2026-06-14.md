# Task 127：多个页面的手机版锚点页面样式调整

## 任务内容

多个页面的手机版本中，有悬浮按钮，点击后如果出现锚点页面，需要去掉半透明黑色背景；锚点页面仍从右侧进入，高度按内容自适应，不再撑满整屏。内容超过屏幕高度时保持屏幕高度并允许内容滚动，关闭按钮在底部展示。

## 修改页面

- `post-detail.html`
- `system-post-detail.html`
- `resource-detail.html`
- `service.html`
- `fastesp-service.html`
- `blog.html`

## Diff 总结

- 修改 `css/detail/post-detail.less`、`css/detail/system-post-detail.less`、`css/detail/resource-detail.less`、`css/detail/service.less`、`css/detail/blog.less` 及对应 `.css` 产物。
- 将移动端锚点面板从全屏半透明遮罩改为右侧白色内容面板。
- 去掉移动端锚点内容的垂直居中伪元素，使内容按自然高度展示。
- 面板高度改为随内容自适应，并保留 `max-height: 100dvh` 及内部滚动能力。
- 面板改为纵向居中显示，右侧与屏幕右侧对齐。
- 增加面板阴影，用于和页面主体内容分割。
- 保留从右侧滑入的动画，并同步动画 transform 以适配纵向居中。
- 处理 `system-post-detail` 页面专属左侧定位覆盖，确保手机版仍从右侧贴边展示。
- 运行 `node scripts/build-pages.js` 生成根目录页面产物，并刷新 CSS/JS 防缓存版本。
