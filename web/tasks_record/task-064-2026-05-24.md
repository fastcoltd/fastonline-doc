`blog.html` 手机版中，右侧信息区默认隐藏，点击浮动按钮后以右侧抽屉方式展示；并修复抽屉顶部内容显示不完整问题。

## 变更摘要
- 仅修改手机版逻辑，PC 端保持原有展示。
- 在 `blog` 页面中复用 `detail-page-menu` 浮动按钮，并将右侧信息区包装为移动端抽屉容器（`#blog-side-panel`）。
- 新增抽屉交互逻辑：
  - 点击浮动按钮打开/关闭抽屉；
  - 点击遮罩或关闭按钮关闭抽屉；
  - 同步 `aria-hidden` / `aria-expanded` 状态；
  - 打开抽屉时执行多时序滚动归零（同步 + `requestAnimationFrame` + `setTimeout`），修复移动端滚动恢复导致的顶部截断。
- 同步样式到 `less/css`：
  - 移动端抽屉遮罩、面板、关闭按钮样式；
  - 抽屉内容容器加入 `-webkit-overflow-scrolling: touch` 与 `overscroll-behavior: contain`；
  - 抽屉内子模块统一 `flex-shrink: 0`，避免首卡片被压缩造成顶部内容显示不完整。
- 执行 `node scripts/build-pages.js`，更新根目录页面与防缓存版本号（`v=20260524141103`）。
- 使用 chrome mcp 实测验证：移动端抽屉打开后顶部展示完整，且“滚动到底->关闭->再打开”能稳定回到顶部。
- 将 `codex_tasks/tasks.md` 对应任务状态更新为“已完成”。

## 影响范围
- 仅影响 `blog.html` 手机版右侧信息区的显示/交互方式及抽屉滚动行为。
- 不影响 PC 端及其他页面功能。
