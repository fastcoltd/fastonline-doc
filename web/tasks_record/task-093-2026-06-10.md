# Task 093 - search-all.html 页面 PC 版分类结果数量调整

## 任务内容

`search-all.html` 页面 PC 版需要修改：选择任意一种分类时只展示该分类第一个 item；选择 All 时只展示各个类型的第一个 item。该修改不能影响移动端。

## 修改内容

- 在 `js/search-all.js` 中新增 PC 端专用的分类卡片映射：
  - Items -> `search-card-first-item-desktop`
  - Campaigns -> `search-card-second-item-desktop`
  - Posts -> `search-card-third-item-desktop`
  - Stores -> `search-card-fourth-item-desktop`
  - Demands -> `search-card-fifth-item-desktop`
- 调整 `filterSearchCardsByType`：
  - PC 端选择 All 时只显示每个类型的第一个 desktop 卡片。
  - PC 端选择单个分类时只显示该分类第一个 desktop 卡片。
  - 移动端继续走原有分类过滤逻辑，避免影响移动端已调整好的展示。
- 有分类结果时隐藏 `no-data-wrapper`，避免有 item 时仍显示空状态。
- 监听窗口 resize，确保 PC/移动端断点切换后按当前 tab 重新过滤。
- 运行 `node scripts/build-pages.js` 生成根目录 HTML，并刷新 css/js 缓存版本号。

## 验证

- `node scripts/build-pages.js`
- `node --check js/search-all.js`
- `node --check scripts/build-pages.js`

## 主要影响文件

- `js/search-all.js`
- `search-all.html`
- 根目录其他 HTML 文件仅由构建脚本刷新 css/js 缓存版本号。
