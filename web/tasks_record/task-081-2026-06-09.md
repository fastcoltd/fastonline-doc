# Task 081 - post-detail.html 手机版锚点弹层居中

## 任务内容

- 页面：`post-detail.html`
- 范围：手机版
- 要求：
  - 点击浮动按钮后出现的锚点页面，内容需要纵向居中显示。
  - 关闭按钮保持原位，不修改。
  - 仅改变内容布局，不修改其他内容。
  - 内容超过一屏高度时，需要可以上下滚动。
  - 检查项目中所有同样样式浮动按钮打开的锚点页面，并按同样要求修改。

## 修改摘要

- 调整移动端锚点弹层内容布局，使用容器级 `::before` / `::after` 弹性占位实现纵向居中。
- 保留弹层内容容器 `overflow-y: auto`，内容超过一屏时可以上下滚动。
- 为目录项补充 `flex-shrink: 0`，避免内容过高时被压缩。
- 关闭按钮 `.table-of-container-close` 的定位未修改。
- 同步检查并覆盖同类锚点弹层页面：
  - `post-detail.html`
  - `resource-detail.html`
  - `service.html`
  - `fastesp-service.html`
  - `blog.html`
- 对 `service.html` 中 `.privacy-policy-page` 的分组目录结构补充同样的容器级居中和滚动规则。
- 对 `blog.html` 中的侧边弹层 `.blog-sidebar-container .other-info-wrapper` 使用同样的容器级居中规则。
- 同步修改对应 `.less` 和 `.css` 文件。
- 执行 `node scripts/build-pages.js` 重新生成根目录页面，并刷新本地 CSS/JS 防缓存版本号。
- 将 `codex_tasks/TASKS.md` 中对应任务状态标记为已完成。
