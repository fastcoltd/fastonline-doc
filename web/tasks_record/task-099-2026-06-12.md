# Task 099：index.html页面的pc版和手机版都需要进行修改

## 任务内容

需要完善购买流程：在已登录的情况下，点击 Best Items 的 buy now 按钮，需要进入购买流程。pc 端以弹框形式呈现，手机端以新页面形式呈现，手机端流程走完后回到 index.html 页面。

## 修改页面

- `index.html`

## Diff 总结

- 新增 `src/partials/components/index-purchase-flow.html`，封装首页购买流程结构，包含商品确认、填写订单需求、确认订单、下单成功等步骤。
- 修改 `src/pages/index.html`，接入购买流程组件，并更新 `css/index.css`、`js/index.js` 防缓存版本。
- 修改 `src/partials/index-best-items-section.html`，为 Best Items 中可购买按钮增加 `index-best-item-buy-btn` 专属入口类名。
- 修改 `js/index.js`，实现首页购买流程交互：
  - 未登录时继续触发登录弹框。
  - 已登录时 pc 端打开购买流程弹框。
  - 已登录时手机端进入页面式购买流程，并在流程完成后返回首页内容。
  - 数量选择同步更新总价和确认信息。
- 修改 `js/common.js`，避免首页新购买入口继续触发旧的全局购买弹框。
- 修改 `css/index.less` 和 `css/index.css`，新增购买流程 pc / mobile 样式；手机端底部操作按钮使用 fixed 固定在底部，内容区域独立滚动，并预留底部空间以兼容一加 Ace5 Pro / Android Chrome。
- 通过 `node scripts/build-pages.js` 同步生成根目录 `index.html`。
