# 无论什么情况下都必须遵守的约定
- always respond in 中文
- 本项目为静态页面项目，所有的网页都是静态的
  - 使用语义化 HTML
  - 使用原生 CSS，不要引入 UI 库
  - 按 Figma 的布局、字号、颜色、间距、圆角实现
  - 优先使用 flex / grid，不要大量 absolute 定位
- 需要注意：css 需要注重复用，避免在各个页面都写了重复的 css，在我给你需求时，先从已有的 css 中查找是否有完全符合要求的 css，如果有，必须复用，如果没有，则新建。如果 css 可以复用，css 的命名最好是通用的命名，以便其他地方复用
- 项目需要适配pc端和手机端，以响应式布局，来调整pc端和手机端展示的样式
- 所有的页面中的内容，宽度都必须限制为 1440，展示在页面横向中央，举个例子，如果屏幕宽度是 1920，那么页面内容只有中间1440的宽度展示内容
- 所有的需求，都需要保证在不修改其他功能的前提下，以最小化修改来实现需求；只修改需求中指定的内容，不要修改其他任何内容
- 所有.css 的修改，需要同步到.less（如果有），保持源码与编译产物一致
- 在修改或者开发完成以后，我需要你通过 chrome mcp 来打开浏览器（需要打开当前用户的浏览器，而不是一个新的浏览器，这样登录信息可以保留，便于调试），检查刚才的修改或者开发的内容，是否1:1 还原了 figma 的设计
- 在完成代码修改后，需要检查是否有语法错误，避免出错
